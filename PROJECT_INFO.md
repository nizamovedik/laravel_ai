## Архитектура проекта
HTTP-запрос
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  1. Маршрут (routes/api.php)                                   │
│     └── Определяет URL, метод, контроллер                     │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. Middleware (app/Http/Middleware/)                         │
│     └── Проверяет: авторизован ли пользователь?               │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. Request (app/Http/Requests/)                              │
│     └── Валидация: корректны ли данные?                       │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. Controller (app/Http/Controllers/)                        │
│     └── Тонкий слой: принимает Request, вызывает Service     │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  5. Service (app/Services/)                                   │
│     └── Бизнес-логика: что делать с данными?                 │
│         ├── Проверка статусов                                 │
│         ├── Транзакции БД                                     │
│         └── Вызов событий (Events)                            │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  6. Repository (app/Repositories/)                            │
│     └── Запросы к БД: как сохранить/получить данные?         │
│         └── Содержит только Eloquent-запросы                  │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  7. Model (app/Models/)                                       │
│     └── Связи, атрибуты, касты (casts), скоупы               │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  База данных (MySQL)                                          │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  8. Event (app/Events/) + Listener (app/Listeners/)          │
│     └── Реакция на события: уведомления, логирование         │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  9. Resource (app/Http/Resources/)                            │
│     └── Форматирование ответа: что вернуть клиенту?          │
└─────────────────────────────────────────────────────────────────┘
    ↓
  Ответ (JSON / Blade)

## Цикл запроса
Как выглядит полный цикл запроса (с примерами)
Пример: создание задачи

1. POST /api/tasks
   ↓
2. middleware: проверка авторизации (auth)
   ↓
3. StoreTaskRequest::rules() → валидация данных
   ↓
4. StoreTaskRequest::toTaskData() → создание DTO
   ↓
5. TaskController::store()
   ├── вызывает $taskService->createTask($taskData)
   ↓
6. TaskService::createTask()
   ├── DB::transaction()
   │   ├── $this->taskRepository->create($data)  ← сохранение в БД
   │   ├── Log::info('...')                      ← логирование
   │   └── event(new TaskCreated($task))         ← событие
   ↓
7. TaskRepository::create()
   ├── Task::create($data)                       ← Eloquent-запрос
   ↓
8. TaskResource($task) → форматирование ответа
   ↓
9. Ответ клиенту: JSON 201
